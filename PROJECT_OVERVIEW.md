# app_v2.2 项目综合说明（答辩与交付版）

## 1. 项目定位与愿景

一句话定位：**把“分散、异构、难解释”的企业数据，转化为“可追溯、可量化、可落地”的投研/风控决策建议。**

面向用户：
- **投资者**：快速完成企业筛选、同业比较与投资建议判断。
- **企业管理者**：定位经营短板，识别风险拖累因子，辅助经营改进。
- **监管与风控机构**：基于结构化证据和可审计评分，提升风险识别透明度。

核心价值主张：
- **确定性 + 智能化**：先用确定性算法给出可复现分数，再让大模型做解释与建议。
- **从“结论”到“证据”闭环**：每个关键判断都能回溯到指标、评分和知识来源。
- **兼顾速度与深度**：简单问数毫秒级响应，复杂分析提供结构化深度报告。

## 3. 核心功能模块详解

### 3.1 智能问数服务（快通道 + 深度分析）

系统接收自然语言问题后，按照“**识别意图 -> 抽取实体 -> 路由执行**”进行处理：
- **快通道**：命中简单指标问句（如“比亚迪2022年销量/营收/ROE”），直接查 `core_metrics_summary`，必要时回退事实表，跳过 RAG/LLM，返回 `chart_type=simple_metric` 的趋势序列。
- 若为复杂分析（如“比亚迪和理想对比，谁更值得投资”），进入证据检索 + 评分聚合 + LLM解说链路，生成完整分析报告。

需求澄清机制：
- 未识别企业名：返回 `needs_clarification`，要求补充企业名/代码。
- 未给时间范围：返回 `needs_clarification`，要求补充年份或“近三年”等窗口。
- 该机制在 `ResponseComposer` 层实现，确保系统不会在关键信息缺失下“硬猜结论”。

### 3.2 确定性与 LLM 双引擎分析（先算后说）

#### A) 评分引擎（确定性）

我们采用“**4大维度 + 多指标线性插值评分 + 动态权重 + ABCD评级**”的可审计模型：

| 维度 | 权重(基础) | 代表指标 |
|---|---:|---|
| 财务健康度 | 0.30 | 流动比率、ROE、现金流覆盖、债务EBITDA等 |
| 行业地位 | 0.25 | 新能源渗透率、渗透率差值、产销偏离等 |
| 法律风险 | 0.25 | 被执行比例、失信次数、商票违约等 |
| 运营能力 | 0.20 | 研发资本化率、自由现金流、担保比例、召回密度等 |

实现细节：
- **线性插值**：指标分段阈值（best/good/ok/bad）间线性计算，不是粗糙打档。
- **动态权重重分配**：当某维度数据完整度低（如 ≤0.3）时，自动降低其权重影响，把权重归一到有效维度。
- **缺失值中性处理**：缺失指标默认中性分（50），并在结果中暴露 `completeness` 和 `confidence`。
- **评级规则**：`A(>=80)`、`B(>=65)`、`C(>=50)`、`D(<50)`，可重复计算、可追踪来源、可审计解释。

这套机制保证：**同样输入 -> 同样输出**，是严肃决策场景最需要的“稳定底座”。

#### B) LLM 智能解说（解释层）

LLM 在本系统中不是“算分器”，而是“**解说员与决策顾问**”：
- 后端把确定性评分、维度得分、指标归因、企业对比快照通过 `scores_json`、`comparison_snapshot` 注入提示词。
- LLM 基于这些结构化输入生成归因与建议，不允许脱离证据“自由发挥”。
- **LLM 超时兜底**：对比查询可设置更短等待（如 20s），超时立即降级为离线结构化报告，保证用户体验阈值。

这种“先算后说”的设计，显著抑制了金融分析场景中大模型常见的幻觉风险。

### 3.3 多维度企业评估与对比（排名 + 表格 + 推荐）

系统支持**多企业**并行对比：
- 证据层并发检索各企业各年份指标与评分。
- 响应层自动构建比较快照，按总分降序输出排名，并生成 Markdown 对比表。
- 前端根据 `chart_type=comparison_ranking` 渲染排名柱状图（第一名金色）+ 雷达图（可用则展示）。

并且系统在结果层强制补全：
- 排名句式（第一/第二/第三...）；
- 推荐建议（`推荐投资：Top1企业`）。

这让“对比结论”从可选项变成必选项，避免报告只罗列数据不落地决策。

### 3.4 风险与机会洞察

归因机制分两层：
- **指标归因**：从指标得分中自动筛选拖累项（低分指标、缺失项），形成 `attribution`。
- **知识归因**：通过 RAG 检索行业研报/新闻/知识文档，补充“为什么会这样”的产业背景解释。

检索链路具备鲁棒性：
- 向量检索（pgvector）为主；
- trigram 文本相似与关键词匹配回退；
- 极端情况下可回退最近文档，保障复杂问题至少有可用知识上下文。

### 3.5 可视化分析大屏（按 chart_type 分流）

前端采用深色科技风，围绕比赛展示需求强化“**专业感 + 堆料感 + 交互感**”：
- 依据后端 `charts.chart_type` 动态渲染：
  - `simple_metric`：指标趋势卡片（折线/柱状）
  - `comparison_ranking`：排名柱状图 + 雷达图（可用时）
  - `legal_risk`：堆积柱状 + 热力分布
- 支持高信息密度图表卡片与可拖拽排序，兼顾演示美观与分析效率。
- 对话区、证据面板、图表面板协同，形成“提问-分析-证据-图表”统一体验。

### 3.6 可信赖报告与证据追溯

系统自动生成结构化报告，至少包含：
- 分析摘要（Summary）
- 关键发现（Key Findings）
- 归因依据（Attributions）
- 决策建议（Recommendations）

每次输出同时携带 `evidence_trail`：
- 证据ID、来源类型、本地/知识库来源、摘要、置信度；
- 支持前端“证据面板”追溯查看。

这意味着报告不再是“黑箱结论”，而是“可核验结论”。

## 4. 数据与知识库底座

### 4.1 数据覆盖广度

项目数据底座以汽车产业链企业为核心，覆盖**269+家企业**的多维数据体系，包含：
- 财务数据（收入、利润、资产、偿债、营运等）
- 销量与新能源渗透相关数据
- 司法与合规风险数据（诉讼、执行等）
- 专利与产业扩展指标（发明、实用新型、外观等）

数据文件说明中明确了“269家企业”主口径，并通过统一单位、统一时间字段、别名映射等规则完成清洗对齐。

### 4.2 知识库（RAG）构建

知识底座由行业研报、新闻、专业文档构成，落地流程为：
1. 文档切片并写入 `documents` 表；
2. 生成向量并存储于 `pgvector` 字段；
3. 查询时按“向量 + 文本相似 + 关键词”多策略融合检索；
4. 将检索片段作为 `knowledge_base` 证据注入分析链路。

由此，系统不仅回答“分数是多少”，还能回答“这个分数背后的行业背景是什么”。

## 5. 性能与技术创新（本次交付关键点）

### 5.1 关键优化点

- **简单查询快速通道**：`core_metrics_summary` + 事实表回退，返回趋势序列，毫秒级响应。
- **对比查询专项提速**：可跳过 RAG + LLM 20 秒超时快速离线兜底，把响应时间控制在体验阈值内。
- **对比结果缓存**：Redis 缓存对比结果（小时级 TTL）提升二次命中速度。
- **多级降级容错**：LLM/RAG/局部数据缺失均可降级，保证系统稳定可演示。

### 5.2 架构演进创新

我们经历了从“多Agent协作”到“单Agent中枢编排”的演进：
- 早期多角色Agent便于探索，但链路复杂、稳定性与可控性不足。
- 当前统一到 `AgentOrchestrator` + `ResponseComposer`，把复杂性收敛在后端编排层。
- 结果是：**更少的架构噪声，更高的工程确定性，更强的答辩可解释性**。

这是一条“从过度设计到极简务实”的工程化升级路径，也是本项目的重要技术思考价值。

## 6. 总结与展望

一句话总结：**本项目把“确定性计算、智能解说、证据追溯、可视化表达”打通为一个可信的企业分析决策闭环。**

未来方向：
- 引入更多实时数据流（公告、舆情、交易、供应链事件）。
- 增强多模态分析能力（图文表混合、时序异常检测）。
- 强化策略仿真能力（场景推演、敏感性分析、风险预警订阅）。

我们希望将系统从“答辩级演示平台”持续演进为“生产级企业经营智能分析中台”：
- 更强实时数据接入（公告/舆情/供应链事件）
- 更强策略仿真（情景推演/敏感性分析/预警订阅）
- 更强多模态表达（图+表+文本一体化解释）
