# 前端 API 层标准化与系统集成测试报告

## 测试环境
- 前端静态服务: `python -m http.server 8002`（目录: `web/`）
- 后端服务: `app_v2` Docker (`http://127.0.0.1:8000`)
- 浏览器基线: Chrome/Edge（手工验证流程按页面交互设计）
- 接口响应标准: `{ code, data, message }`

## 本轮改造范围
- 新增统一客户端: `web/js/api-client.js`
- 登录页接入真实 API: `web/login.html`, `web/js/login.js`
- 主页面改造为统一 API 调用: `web/js/app.js`, `web/index.html`
- 后端联调修复: `app_v2/app/main.py`（启动自愈 `users` 表）

## 用例结果
| 测试场景 | 操作步骤 | 预期结果 | 结果 |
| :--- | :--- | :--- | :--- |
| 注册新用户 | 登录页输入账号密码，点击注册 | 注册成功并可进入系统 | 通过（API 200） |
| 登录 | 使用刚注册账号登录 | 跳转主页面，显示用户信息 | 通过（API 200，`getMe` 200） |
| 评分查询 | 输入“比亚迪 2022 年评分” | 展示总分、评级、维度分 | 通过（`/api/v1/scoring/{stock}/{year}` 200） |
| Agent 分析 | 输入“分析比亚迪 2022 年财务风险” | 返回结构化报告+图表 | 通过（`status=completed`） |
| 澄清机制 | 输入“比亚迪的营收” | 返回澄清问题 | 通过（`status=needs_clarification`） |
| 退出登录 | 点击退出按钮 | 清除 token 并回登录页 | 通过 |

## 关键验证细节
- 前端页面可访问:
  - `http://127.0.0.1:8002/login.html` => `200`
  - `http://127.0.0.1:8002/index.html` => `200`
- 后端链路自动化检查:
  - 注册/登录/获取用户/评分/Agent 分析/澄清均返回成功
  - 无 500 错误
- 401 处理:
  - `ApiClient` 遇 401 会清理 token 并跳转登录页
- 网络健壮性:
  - `ApiClient` 网络失败自动重试 2 次（间隔 1s）
- 竞态控制:
  - 聊天发送期间禁用重复提交，避免快速连续发送导致结果串扰

## 发现的问题与修复记录
1. **前端请求分散且直接 `fetch`**
   - 修复: 全部改为 `ApiClient` 统一调用，业务层不再直接 `fetch`。
2. **登录页硬编码旧接口**
   - 修复: 改为调用 `apiClient.login/register`，并加基础表单校验。
3. **后端 `users` 表缺失导致认证 500**
   - 修复: 在 `app_v2` 启动阶段增加 `users` 表与索引自愈创建。
4. **本地静态部署路径兼容性问题**
   - 修复: `index.html`/`login.html` 脚本引用改为相对路径，兼容 `/web/` 与根路径静态托管。

## 风险说明
- 当前 `app_v2` 进程未直接挂载 `web` 到 `http://127.0.0.1:8000/web/*`（该路径返回 404），本次演示采用 `8002` 独立静态服务 + `8000` API 联调。
- 文件上传接口在 `app_v2` 尚未开放，`ApiClient.uploadFile()` 已做明确错误提示与降级处理。

## 最终结论
前端已完成对新后端核心能力（认证、评分、Agent）的统一 API 对接，关键演示链路可稳定跑通，**可对外演示**。  
若要实现“单端口一体化部署”，建议下一步在 `app_v2` 中补充静态目录挂载或 Nginx 统一反向代理。
